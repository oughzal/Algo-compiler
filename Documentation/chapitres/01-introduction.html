📚 Chapitre 1 : Introduction à l'Algorithmique

Qu'est-ce qu'un algorithme ?

Un algorithme est une suite finie et ordonnée d'instructions qui permet de résoudre un problème ou d'accomplir une tâche. C'est comme une recette de cuisine : vous suivez des étapes précises pour obtenir un résultat.

Exemple de la vie quotidienne

Algorithme pour faire du thé :

  1. Faire bouillir de l'eau
  2. Mettre un sachet de thé dans une tasse
  3. Verser l'eau chaude dans la tasse
  4. Laisser infuser 3 minutes
  5. Retirer le sachet de thé
  6. Ajouter du sucre si désiré

Caractéristiques d'un bon algorithme

Un algorithme efficace doit posséder les caractéristiques suivantes :

Caractéristique Description
Fiabilité Produit toujours le résultat attendu pour des entrées valides
Clarté Facile à comprendre et à lire
Efficacité Utilise un minimum de ressources (temps, mémoire)
Généralité Résout un problème général, pas seulement un cas particulier
Finitude Se termine toujours après un nombre fini d'étapes

Structure d'un algorithme en Algo

Dans le langage Algo, chaque algorithme suit une structure bien définie. Voici la structure minimale :

Structure de base
algorithme NomDeLAlgorithme

variables
    // Déclaration des variables

debut
    // Instructions de l'algorithme
fin

Détails de la structure

  • algorithme : Mot-clé obligatoire suivi du nom de l'algorithme
  • variables : Section où l'on déclare les variables (optionnelle)
  • debut : Marque le début des instructions
  • fin : Marque la fin de l'algorithme

Premier exemple : "Bonjour le monde"

Voici votre premier algorithme en Algo. Il affiche simplement un message à l'écran :

bonjour.algo
algorithme BonjourMonde

debut
    ecrireln("Bonjour le monde !")
fin
Résultat de l'exécution
Bonjour le monde !

Explication ligne par ligne

Ligne Explication
algorithme BonjourMonde Déclare un algorithme nommé "BonjourMonde"
debut Début du corps de l'algorithme
ecrireln("Bonjour le monde !") Affiche le texte "Bonjour le monde !" suivi d'un retour à la ligne
fin Fin de l'algorithme

Exemple avec interaction

Un algorithme plus intéressant qui demande le nom de l'utilisateur et le salue :

salutation.algo
algorithme Salutation

variables
    nom : chaine
    age : entier

debut
    // Demander le nom
    ecrire("Comment vous appelez-vous ? ")
    lire(nom)

    // Demander l'âge
    ecrire("Quel âge avez-vous ? ")
    lire(age)

    // Afficher un message personnalisé
    ecrireln()
    ecrireln("Bonjour ", nom, " !")
    ecrireln("Vous avez ", age, " ans.")

    // Condition simple
    si age >= 18 alors
        ecrireln("Vous êtes majeur.")
    sinon
        ecrireln("Vous êtes mineur.")
    finsi
fin
Exemple d'exécution
Comment vous appelez-vous ? Alice
Quel âge avez-vous ? 20

Bonjour Alice !
Vous avez 20 ans.
Vous êtes majeur.

Nouveaux concepts introduits

  • Variables : nom (de type chaîne) et age (de type entier)
  • Entrée : lire() pour récupérer des données de l'utilisateur
  • Sortie : ecrire() et ecrireln() pour afficher des informations
  • Commentaires : Lignes commençant par // (ignorées lors de l'exécution)
  • Condition : si...alors...sinon pour faire des choix

Les commentaires

Les commentaires permettent d'expliquer le code. Ils sont ignorés lors de l'exécution et servent uniquement à la documentation.

Exemples de commentaires
algorithme ExempleCommentaires

debut
    // Ceci est un commentaire sur une ligne
    ecrireln("Hello")  // Commentaire en fin de ligne

    # On peut aussi utiliser le dièse pour les commentaires
    ecrireln("World")
fin
Bonne pratique

Commentez votre code pour expliquer pourquoi vous faites quelque chose, pas ce que vous faites (si c'est évident). Un code bien écrit devrait être auto-documenté.

Conventions de nommage

Pour rendre votre code plus lisible, suivez ces conventions :

Élément Convention Exemples
Algorithme PascalCase (majuscule à chaque mot) CalculerMoyenne, TrierTableau
Variables camelCase (minuscule puis majuscules) nomUtilisateur, ageEtudiant
Constantes MAJUSCULES_AVEC_UNDERSCORES PI, TAUX_TVA
Fonctions camelCase avec verbe calculerSomme, afficherResultat

Exercices

Exercice 1 : Présentation

Écrivez un algorithme qui demande à l'utilisateur son nom, son prénom et sa ville, puis affiche une phrase de présentation complète.

Exemple de sortie :

Je m'appelle Alice Dupont et j'habite à Paris.
Exercice 2 : Calcul simple

Écrivez un algorithme qui demande deux nombres à l'utilisateur et affiche leur somme, leur différence, leur produit et leur quotient.

Exercice 3 : Année de naissance

Écrivez un algorithme qui demande l'âge de l'utilisateur et calcule son année de naissance approximative (en supposant que nous sommes en 2026).

Points clés à retenir

Résumé du chapitre
  • Un algorithme est une suite d'instructions pour résoudre un problème
  • La structure de base : algorithme → variables → debut → fin
  • ecrire() et ecrireln() pour afficher
  • lire() pour récupérer des données
  • Les commentaires avec // ou #
  • Importance des conventions de nommage
